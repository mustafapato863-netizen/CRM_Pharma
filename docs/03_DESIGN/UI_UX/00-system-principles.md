# System Principles

## Product Direction

Pharmacy CRM should feel like a premium operational SaaS, not a classic ERP.
The reference language is:
- Linear
- Stripe Dashboard
- Notion
- Vercel
- Raycast
- Odoo

## Visual Rules

- Light surfaces first.
- Glass only where it adds depth.
- Soft gradients, never loud color blocks.
- High whitespace control, not oversized empty space.
- Clear hierarchy between primary, secondary, and tertiary actions.
- One component language across the whole app.

## What To Remove

- Duplicate headers
- Repeated filters
- Decorative sections without business value
- Fake notifications
- Hero blocks that do not help a daily workflow
- Dark form surfaces unless absolutely necessary

## What To Keep

- Fast scanning
- Dense tables
- Strong form labels
- Business alerts
- Consistent empty/error states
- Keyboard-friendly interactions

## Typography

- Use a clean sans system.
- Titles should be bold and compact.
- Supporting text should be calm and readable.
- Keep Arabic and English weights visually balanced.

## Interaction Style

- Hover should add clarity, not noise.
- Loading should feel structured, not blank.
- Success and validation should be immediate.
- Animations should be subtle and fast.
