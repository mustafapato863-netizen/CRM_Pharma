# Page Map

This is the page-by-page UX reference for the rebuild.

## Core Pages

### Dashboard
- Role: operational overview
- Primary task: identify what needs attention
- Do: show KPIs, alerts, and quick actions
- Avoid: too much narrative copy or repeated hero sections

### Products
- Role: catalog management
- Primary task: search, inspect, create, edit
- Do: dense table, strong filters, clear status badges
- Avoid: oversized cards or multiple competing toolbars

### Batches
- Role: stock lot control
- Primary task: manage expiry-aware inventory
- Do: product filter, expiry/status emphasis, table-first layout
- Avoid: unnecessary decorative sections

### Parties
- Role: customer and supplier management
- Primary task: find, create, update, and review balances
- Do: workspace summary, filters, table, drawer form
- Avoid: separate flows for customers and suppliers unless needed

### Stock
- Role: stock IN / stock OUT operational shell
- Primary task: fast movement entry
- Do: mode-aware header, compact controls, clear entry panel
- Avoid: mixing review and entry too heavily

### Inventory Movements
- Role: history and audit
- Primary task: filter and review movements
- Do: table-first workspace with export and print
- Avoid: making it behave like a second entry screen

### Purchases
- Role: purchase invoice creation
- Primary task: record supplier invoices and receive stock
- Do: fast invoice flow, batch creation where needed

### Sales
- Role: sales invoice creation
- Primary task: record customer invoices and issue stock
- Do: FEFO-aware flow, review before submit

### Payments
- Role: cash movement tracking
- Primary task: record receipts and payments
- Do: table plus focused form

### Returns
- Role: correction workflow
- Primary task: reverse or adjust previous movement
- Do: clear reason and reference handling

### Ledger
- Role: account review
- Primary task: inspect balances and transaction history
- Do: timeline/table with clear totals

### Reports
- Role: analytics
- Primary task: quickly answer business questions
- Do: cards, filters, export, print

### Expiry Alerts
- Role: risk monitoring
- Primary task: catch soon-to-expire items
- Do: alert-first layout with direct action links

### Users
- Role: access and permission management
- Primary task: manage accounts, roles, and security
- Do: table or list with strong status signals

### Settings
- Role: configuration
- Primary task: update system and pharmacy preferences
- Do: sectioned cards and explicit save/reset buttons

### Backup
- Role: maintenance
- Primary task: export, restore, and protect data
- Do: action-driven layout, not a generic page

## Support Pages

### Login
- Role: entry point
- Primary task: authenticate quickly
- Do: one strong panel, language switcher, minimal distractions

### Maintenance
- Role: system holding state
- Primary task: explain downtime cleanly

### Not Found
- Role: recovery
- Primary task: guide the user back to a valid route
