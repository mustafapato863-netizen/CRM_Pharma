# Database Schema

## Purpose

Documents the simplified MVP data model used for stock, balances, due payments, and expiry tracking.

## Responsibilities

- Explain the core entities and relationships.
- Document data ownership and lifecycle rules.
- Keep schema decisions understandable before and after migrations exist.

## What Should Be Documented Here

- Entity descriptions.
- Field definitions.
- Relationship rules.
- Indexing strategy.
- Uniqueness constraints.
- Audit fields.

## MVP Entities

- `User`
- `Product`
- `Party`
- `StockMovement`
- `LedgerEntry`
