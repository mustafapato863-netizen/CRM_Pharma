# دليل تهيئة وتشغيل النظام في بيئة الإنتاج (Production Deployment Guide)

يحتوي هذا الملف على المتطلبات والخطوات التفصيلية لنشر وتشغيل **نظام الصيدلية وإدارة المستودعات (Pharmacy CRM)** بنجاح في بيئة الإنتاج الفعلية.

---

## 1. المتطلبات الأساسية (Prerequisites)

قبل البدء بالتشغيل، تأكد من توفر الآتي على الخادم (Server):
* **Node.js**: إصدار 18.0.0 أو أحدث (يوصى بـ LTS).
* **PostgreSQL Database**: خادم قاعدة بيانات نشط ومهيأ للاستقبال.
* **npm**: مدير الحزم (يأتي تلقائياً مع Node.js).
* **PM2**: لإدارة تشغيل التطبيق في الخلفية وضمان استمرارية العمل (اختياري ولكنه يوصى به بشدة).

---

## 2. خطوات التجهيز والتشغيل (Deployment Steps)

### الخطوة 1: تثبيت حزم الاعتماديات
قم بتنزيل الحزم النظيفة الخاصة ببيئة التشغيل بالكامل:
```bash
npm ci --only=production
# أو للتثبيت العادي:
npm install
```

### الخطوة 2: تهيئة ملف البيئة (.env)
قم بإنشاء ملف `.env` في المجلد الرئيسي للمشروع بناءً على النموذج المرفق في `.env.example`:
```bash
cp .env.example .env
```
وقم بتعديل المتغيرات التالية:
1. `DATABASE_URL`: رابط الاتصال بقاعدة بيانات PostgreSQL الفعلية.
2. `AUTH_SECRET` و `NEXTAUTH_SECRET`: مفاتيح تشفير عشوائية قوية لتأمين جلسات المستخدمين (يمكنك توليدها عبر تشغيل الأمر: `openssl rand -base64 32`).
3. `NEXTAUTH_URL`: الرابط الفعلي للموقع (مثال: `https://pharmacy.yourdomain.com`).
4. `API_AI_KEY`: مفتاح الـ API لخدمة الذكاء الاصطناعي (Gemini أو OpenAI).

### الخطوة 3: إنشاء وتطبيق جداول قاعدة البيانات (Prisma Migrations)
قم بتشغيل الأمر التالي لإنشاء الجداول في قاعدة البيانات الفعلية دون التأثير على البيانات الحالية:
```bash
npx prisma migrate deploy
```
ثم قم بتوليد مكتبة استعلامات قاعدة البيانات (Prisma Client):
```bash
npx prisma generate
```

*(اختياري)* إذا كنت تريد إدخال بيانات تجريبية أولية (Seed Data) لتجربة النظام بالكامل:
```bash
npx prisma db seed
```

### الخطوة 4: بناء التطبيق للإنتاج (Production Build)
قم بعملية تجميع وبناء كود المشروع لبيئة الإنتاج للحصول على أقصى سرعة وأداء:
```bash
npm run build
```

## 3. تشغيل وإدارة الخدمة (Process Management)

يوصى باستخدام **PM2** لتشغيل التطبيق في الخلفية ليقوم بإعادة التشغيل الذاتي في حال حدوث أي توقف أو عند إعادة تشغيل السيرفر بالكامل.

### التشغيل باستخدام PM2:
قم بتثبيت PM2 عالمياً إذا لم يكن ثابتاً:
```bash
npm install -g pm2
```

قم ببدء تشغيل النظام كخدمة مستمرة:
```bash
pm2 start npm --name "pharmacy-crm" -- start -- -p 3000
```

### لحفظ الحالة الحالية لتبدأ الخدمة تلقائياً عند إقلاع السيرفر:
```bash
pm2 save
pm2 startup
```

### أوامر التحكم المفيدة:
* **عرض حالة الخدمة**: `pm2 status`
* **متابعة سجل الأخطاء (Logs)**: `pm2 logs pharmacy-crm`
* **إعادة تشغيل النظام**: `pm2 restart pharmacy-crm`
* **إيقاف الخدمة**: `pm2 stop pharmacy-crm`

## 4. إعدادات جدار الحماية والبروكسي العكسي (Reverse Proxy)

لتأمين الاتصال عبر بروتوكول **HTTPS** وتوجيه المنفذ 80/443 إلى المنفذ الافتراضي للنظام (3000)، يوصى بإعداد **Nginx** كبروكسي عكسي.

نموذج تهيئة Nginx (`/etc/nginx/sites-available/default`):
```nginx
server {
    listen 80;
    server_name pharmacy.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 5. خطة النسخ الاحتياطي (Backup Strategy)
* يحتوي النظام على ميزة نسخ احتياطي مضمنة في صفحة **الإعدادات (`/settings`)** تمكنك من تصدير واستيراد قاعدة البيانات بالكامل كملفات SQL مشفرة بضغطة واحدة.
* يوصى بجدولة نسخ احتياطي يومي لقاعدة البيانات PostgreSQL على مستوى نظام التشغيل عبر `pg_dump` لضمان أمان البيانات المطلق.
